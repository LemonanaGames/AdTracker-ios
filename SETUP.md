# Mac-side setup & validation

The full app is implemented across all phases on Windows; the following must be done on a Mac
with Xcode 26 to build, sign and validate. The app **runs end-to-end on mock/cached data** with
none of the credential steps — those only unlock live data, IAP, and AI.

## 1. Generate & build
```sh
brew install xcodegen
cd AdReport && xcodegen generate
open AdReport.xcodeproj
```
- Resolve Swift Packages (RevenueCat) when prompted.
- Schemes: **AdReport** (app + widgets + tests) and **AdReportWatch** (watch app + complications).
- Run **AdReport** on an iOS 26 simulator; run `AdReportTests` for the data-layer suite.

## 2. Signing & capabilities (per target)
Enable **App Groups** = `group.com.millionappz.adreport` on: `AdReport`, `AdReportWidgets`,
`AdReportWatch`, `AdReportWatchWidgets` (entitlement files are generated by XcodeGen; the App Group
must also exist in the Apple Developer portal and be in each provisioning profile).
Set your DEVELOPMENT_TEAM in `project.yml` `settings.base` (add `DEVELOPMENT_TEAM: <TEAMID>`).

## 3. RevenueCat (paywall / Reporting Pro) — optional
- Add products (Lifetime $29.99, Annual $19.99) in App Store Connect; create a RevenueCat offering
  with entitlement id **`pro`**.
- Set `Secrets.revenueCatAPIKey` in `AdReport/Services/SubscriptionStore.swift`.
- Until set, the paywall unlocks Pro locally (demo).

## 4. AdMob live data — optional
- Create a Google Cloud OAuth client (iOS) and enable the **AdMob API**.
- Set `AdMobConfig.clientID` in `AdReport/Services/AdMobAuth.swift`.
- Add the **reversed client id** as a URL scheme (extend `CFBundleURLSchemes` in `project.yml`).
- Validate the `AdMobClient` response parsing (dimension/metric enum names) against your real report.

## 5. AppLovin live data
- In-app: **Settings → Accounts & Networks** (or Add account) → paste your **Report Key**.
- Validate `AppLovinClient` column names against your account's `maxReport` JSON.

## 6. Watch app embedding
The watch targets currently build on their own scheme. To ship the watch app with the phone app,
add `AdReportWatch` as an embedded dependency of the `AdReport` target (the
`WKCompanionAppBundleIdentifier` is already set).

## 7. App icon
`Resources/Assets.xcassets/AppIcon.appiconset` is an empty placeholder — drop in a 1024² icon.

## TestFlight (automated via `.github/workflows/testflight.yml`)

CI (`ios.yml`) builds + tests on every push. To ship to TestFlight, run the **Deploy to TestFlight**
workflow manually (Actions tab → Run workflow). One-time setup:

1. **App record:** create an app for bundle id `com.millionappz.adreport` in App Store Connect.
2. **App Group:** create `group.com.millionappz.adreport` and assign it to the app IDs
   (`…adreport`, `.widgets`; add `.watchkitapp*` if you ship the watch app).
3. **App Store Connect API key:** Users & Access → Integrations → create a key (App Manager role).
4. **Repo secrets** (Settings → Secrets and variables → Actions):
   `APPLE_TEAM_ID`, `ASC_KEY_ID`, `ASC_ISSUER_ID`, `ASC_KEY_P8_BASE64`,
   `DIST_CERT_P12_BASE64` (your Apple Distribution cert exported as .p12), `DIST_CERT_PASSWORD`.

The workflow archives the `AdReport` scheme (app + widgets), auto-manages provisioning with the API
key, exports an `.ipa`, and uploads it to TestFlight (build number = the run number). The watch app
ships separately once embedded (see step 6 above).

> I can't run this from Windows — it needs your signing secrets and the app record. Once the secrets
> are in, it's a one-click deploy from the Actions tab.

## Known assumptions to verify
- AdMob/AppLovin JSON shapes (parsing is defensive but column names should be confirmed).
- Foundation Models (`InsightEngine`) only runs on Apple-Intelligence devices; otherwise the
  Smart-Insights card uses the deterministic rule-based fallback.
- Channel-push widget/Live-Activity updates are not yet wired (local updates on sync are).
